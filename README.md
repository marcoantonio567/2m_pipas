# 2M Pipa System

Django web system for managing a kite store, with customer registration, products, sales, financial cash management, and dashboards.

## Features

- Main menu with access to core modules.
- Dashboard with indicators for sales, revenue, expenses, gross profit, and net profit.
- Customer registration, editing, listing, and deletion.
- Product registration, editing, listing, and deletion with inventory control, cost price, selling price, and profit tracking.
- Sales recording with customer, product, quantity, and payment method.
- Automatic inventory deduction when recording a sale.
- Automatic inventory return when deleting a sale.
- Automatic creation of incoming financial transactions for each sale.
- Financial cash management with income, expenses, balance, and pagination.
- Financial categories registration and management.
- Command to populate the database with dummy data.

## Technologies

- Python
- Django
- SQLite
- HTML and CSS

## How to run locally

1. Create and activate a virtual environment:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

2. Install Django:

```powershell
pip install -r requirements.txt
```

3. Apply migrations:

```powershell
python manage.py migrate
```

4. Optionally, load dummy data to test the system:

```powershell
python manage.py seed_mock_data
```

To clean and recreate dummy data:

```powershell
python manage.py seed_mock_data --reset-mock
```

To remove all application data from the database:

```powershell
python manage.py clear_database
```

To run the cleanup without the confirmation prompt:

```powershell
python manage.py clear_database --yes
```

5. Start the development server:

```powershell
python manage.py runserver
```

6. Access in your browser:

```text
http://127.0.0.1:8000/
```

## Notes

- The project uses SQLite by default locally and PostgreSQL on Vercel.
- Templates and static files are located in the `frontend/` folder.
- The `venda` (sale) financial category is protected by the system as it is used to automatically record income generated by sales.
- Before publishing to production, configure `SECRET_KEY`, `DEBUG`, `ALLOWED_HOSTS`, `CSRF_TRUSTED_ORIGINS`, and database variables.

## Deploy on Vercel

1. Create a PostgreSQL database, for example on Supabase.
2. In Vercel, add these environment variables:

```text
SECRET_KEY=replace_with_a_secure_random_secret
DEBUG=False
DATABASE_ENGINE=postgres
SUPABASE_DB_URL=postgresql://postgres.your-project-ref:url_encoded_password@aws-1-us-west-2.pooler.supabase.com:6543/postgres
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_KEY=your_supabase_anon_key
ALLOWED_HOSTS=.vercel.app,your-domain.com
CSRF_TRUSTED_ORIGINS=https://your-project.vercel.app,https://your-domain.com
```

For Supabase pooler connections on port `6543`, use the pooler user format `postgres.<project-ref>`. If your database password has special characters, URL-encode them inside `SUPABASE_DB_URL`, for example `@` as `%40` and `#` as `%23`. Avoid keeping an old conflicting `DATABASE_URL` in Vercel; this project prefers `SUPABASE_DB_URL` when both are present.

3. Deploy the repository on Vercel. The `vercel.json` file selects the Django preset and collects static files for the Vercel CDN.
4. Apply migrations against the production database:

```powershell
python manage.py migrate
```

Run that command with the same production database environment variables configured locally, or from a secure one-off environment.
